## What Went Well

### Technical Achievements

1. **Anomaly Engine Architecture**
   - `anomalyEngine.ts` contient uniquement des fonctions pures (pas de side effects)
   - Testabilité maximale avec 49 tests unitaires
   - Constantes exportées pour configurabilité (CWV_THRESHOLDS, SEVERITY_COEFFICIENTS)

2. **Three-Layer Anomaly Detection**
   - **Threshold anomalies:** Dépassements seuils absolus Google CWV
   - **Delta anomalies:** Changements significatifs M/M-1 (>10%)
   - **Trend anomalies:** Dégradation continue sur 3 mois

3. **Vulnerability Scoring System**
   - Score = âge (jours) × coefficient sévérité (critical=4, high=3, medium=2, low=1)
   - Tri automatique par score décroissant
   - Priorisation des vulnérabilités anciennes et critiques

4. **TDD Implementation**
   - Tests écrits avant le code
   - vi.useFakeTimers() pour contrôle du temps dans les tests
   - Couverture des edge cases (données insuffisantes, valeurs nulles)

5. **UI Components**
   - `InvestigateBanner` avec affichage conditionnel (v-if anomalies.length > 0)
   - `AnomalyCard` avec badges sévérité colorés
   - Navigation vers la page d'axe concernée
   - Transition animée expand/collapse

## Challenges & Learnings

### Technical Decisions

1. **Pure Functions in anomalyEngine.ts**
   - Aucun accès DB, aucun side effect
   - Facilite le testing et la réutilisation
   - Composable `useAnomalies` orchestre l'accès aux données

2. **Inverted Metrics for Trend Detection**
   - Réutilisation de `metricPolarity.ts` (Story 3.1)
   - Dégradation = augmentation pour LCP/INP/erreurs, diminution pour coverage

3. **Severity Badge Colors**
   - critical → red
   - warning → amber
   - info → blue
   - Utilisation des tokens Nuxt UI (pas de couleurs hardcodées)

4. **Expand/Collapse Animation**
   - Transition CSS native pour performance
   - État d'expansion local au composant InvestigateBanner

### Areas for Improvement

1. **Threshold Configuration**
   - Actuellement hardcodé dans `CWV_THRESHOLDS`
   - Future : configuration via env vars ou admin UI

2. **Anomaly Persistence**
   - Anomalies recalculées à chaque chargement
   - Future : caching ou stockage en DB pour historique

### Technical Debt

| Item | Severity | Description |
|------|----------|-------------|
| Hardcoded thresholds | Low | CWV thresholds in code, not configurable |
| No anomaly history | Low | Anomalies not persisted for trending |
