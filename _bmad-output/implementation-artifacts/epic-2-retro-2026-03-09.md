## Challenges & Learnings

### Code Review Findings (2026-03-09)

1. **CR-2: ZIP Parsing Fragile (Story 2.3)**
   - `parseCoverageFromZip` uses regex on ZIP binary decoded as text
   - Works for lightly compressed ZIPs; may fail on heavily compressed ones
   - **Decision:** Acceptable for MVP — monitor in production
   - **Action:** If issues arise, implement proper JSZip or @std/archive

2. **CR-3: Grammar Error Fixed (Story 2.2)**
   - `formatAge` function had a grammatical issue → corrected

3. **CR-4: Collector Tests = Value Tests (All Stories)**
   - Unit tests verify transformation logic, not actual API calls
   - Tests mock external APIs and validate mapping functions
   - **Decision:** Appropriate for MVP — integration tests would require API mocking complexity

### Areas for Improvement

1. **API Key Management**
   - Multiple API keys now required (DEBUGBEAR_API_KEY, GITHUB_TOKEN)
   - `.env.example` updated but production configuration pending
   - **Action:** Configure Supabase Edge Function secrets before deployment

2. **pg_cron Configuration**
   - Jobs defined in stories but not yet scheduled in production
   - **Action:** Apply pg_cron schedules (weekly DebugBear, daily Dependabot/Coverage, 15min GitHub)

### Technical Debt

|------|----------|-------------|
| CR-2: ZIP parsing | Low | Regex-based ZIP parsing works for now, may need JSZip later |
