## What Went Well

### Technical Achievements

1. **Metric Polarity System**
   - `metricPolarity.ts` centralise la logique des métriques inversées (lower is better)
   - Couleurs sémantiques automatiques : amélioration=vert, dégradation=rouge
   - DeltaBadge gère automatiquement l'inversion

2. **ECharts Advanced Features**
   - M/M-1 overlay avec série pointillée (z-index, opacité)
   - markLine pour les annotations déploiements
   - Tooltip custom pour les marqueurs MEP

3. **Backfill Architecture**
   - Interface commune `BackfillParams` pour tous les collecteurs
   - API Nitro `backfill.post.ts` pour déclencher le backfill
   - Gestion élégante des APIs sans historique (Dependabot → status 'partial')

4. **State Management**
   - `useRepository` avec `useState` pour SSR compatibility
   - Repo sélectionné persiste dans le state global
   - Toutes les queries filtrent par `repository_id`

5. **Deep Links**
   - `deepLinks.ts` génère les URLs pour Sentry, GitHub, DebugBear
   - Ouverture nouvel onglet avec `target="_blank"` et `rel="noopener noreferrer"`
   - Navigation < 2 clics respectée

## Challenges & Learnings

### Technical Decisions

1. **Chart Date Alignment**
   - M-1 dates shifted to align visually with current period
   - Décision : overlay utilise les mêmes positions X, légende indique M-1

2. **Backfill Limitations**
   - Sentry : max 90 jours (API limit)
   - Dependabot : pas de vrai historique, snapshot uniquement
   - Coverage : dépend de la rétention des artifacts GitHub (~90j)
   - **Decision:** Log status 'partial' avec warning explicite

3. **GitHub Backfill Pagination**
   - Safety limit : MAX_BACKFILL_PAGES=50
   - Évite les boucles infinies sur gros historiques

4. **Date Validation**
   - JS Date accepte Feb 30 → crée March 2
   - Ajout de `isValidISODate()` avec validation stricte

### Areas for Improvement

1. **Backfill UX**
   - Actuellement trigger manuel via API
   - Future : bouton dans l'UI Health page

2. **Multi-repo Support**
   - RepoSelector en place mais un seul repo actif (MVP)
   - Prêt pour V1.5 multi-repo

### Technical Debt

| Item | Severity | Description |
|------|----------|-------------|
| Backfill manual only | Low | Pas de UI pour trigger backfill |
| Dependabot no history | Architectural | API limitation, not fixable |
