## Challenges & Learnings

### Technical Decisions

1. **Client-Side Generation**
   - Toute la génération Markdown se fait côté client
   - Données déjà disponibles via useReport composable
   - Aucun round-trip serveur nécessaire

2. **Timezone Handling**
   - Fix appliqué pour getMonthStart/getMonthEnd (UTC)
   - Fix appliqué pour formatPeriodDisplay (UTC)
   - Évite les décalages de date entre client et serveur

3. **String Concatenation vs Template**
   - Génération par concaténation de strings (pas de lib externe)
   - Plus simple et plus contrôlable pour le formatage Markdown

4. **Trend Detection**
   - Analyse des deltas pour déterminer la tendance globale
   - improving : majorité des métriques s'améliorent
   - degrading : majorité des métriques se dégradent
   - stable : pas de tendance claire

### Code Review Fixes

- `cacheKey` changé en fonction (évite le stale cache)
- Suppression de `rawMetrics` query non utilisée
- Timezone UTC pour les dates

### Areas for Improvement

1. **PDF Export**
   - Markdown uniquement pour MVP
   - Future : export PDF pour les stakeholders non-tech

2. **Email Integration**
   - Future : envoi automatique du rapport par email
